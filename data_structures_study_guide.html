Lecture 4 & 5 — Linked Lists
Core concept

What is a linked list?

A sequence of nodes where each node holds data + a pointer to the next node. Unlike arrays, memory is allocated dynamically — no need to declare a fixed size in advance.

STARTPointer to the first node. If START = NULL, the list is empty.
NULLMarks the end of the list (stored in the last node's pointer).
PTRA temporary pointer used to traverse the list node by node.
Advantages
  • Dynamic size — grows/shrinks at runtime
  • Efficient memory use (allocate only what's needed)
  • Easier insertions and deletions than arrays
Disadvantages
  • Extra memory needed for pointer in each node
  • Searching is slow — must traverse one by one
  • No direct/random access like arrays
Types of linked lists

Singly Linked List

Each node has one pointer → next node only. Traversal is one direction only.

Circular Linked List

Like singly, but the last node's pointer holds the address of the first node instead of NULL — forms a circle. Can start traversal from any node.

Doubly Linked List

Each node has three parts: previous pointer | data | next pointer. Can be traversed in both directions. First node's previous = NULL, last node's next = NULL.

Header Linked List

Always starts with a special header node that stores info about the list (e.g. total count, sorted status). Two variants: Grounded (last node → NULL) and Circular (last node → header).

Operations — Singly Linked List
TraverseVisit each node once using PTR starting from START until NULL.
SearchWalk the list comparing each node's data to SEARCH_VAL. Returns address if found.
InsertAt beginning, at end, or after a given node. Always check for overflow first (is memory available?).
DeleteFrom beginning, end, or after a given node. Always check for underflow first (is list empty? START = NULL?).
ConcatenateJoin two linked lists into one.
Sort/ReverseRearrange nodes in order; reversing uses three pointers: PREV, PTR, NEW.
Application

Polynomial representation

Each term of a polynomial (e.g. 10x² + 6x + 9) is stored as a node with two fields: coefficient and power. Linked lists can then perform operations on polynomials.

Lecture 6 & 7 — Queues
Core concept

What is a queue?

A FIFO (First In, First Out) data structure. Elements enter at the REAR and leave from the FRONT. Think of a line at a movie theater — first person in line is served first.

REARWhere new elements are inserted (the tail).
FRONTWhere elements are removed (the head).
Array implementation

FRONT and REAR both start at position 0. As elements are inserted, REAR increments. Deletion moves FRONT. Best when max queue size is known in advance.

Linked list implementation

Uses dynamic memory — START pointer becomes FRONT, end of list becomes REAR. Best when size is unknown. All insertions at REAR, all deletions at FRONT.

Basic operations
EnqueueInsert at REAR. Check overflow first (REAR = MAX – 1 means queue is full).
DequeueRemove from FRONT. Check underflow first (FRONT > REAR or FRONT = –1 means queue is empty).
Types of queues

Circular Queue

Connects the last position back to the first — solves the wasted-space problem of linear queues. New positions calculated as: REAR = (REAR + 1) % MAX and FRONT = (FRONT + 1) % MAX. Also called a "ring buffer."

Priority Queue

Each element has a priority. Higher priority elements are processed first. Equal priority → First Come First Served. Used in OS process scheduling.

  • Ascending: smallest element deleted first
  • Descending: largest element deleted first

In linked representation, each node has 3 parts: data | priority number | next address.

De-queue (Double-ended Queue)

Insertion and deletion can happen at either end. Uses LEFT and RIGHT pointers. Implemented using circular arrays. Also called a head-tail linked list.

Real-world applications
  • Call center phone systems — callers held in order
  • Real-time interrupt handling — First Come First Served
  • CPU round-robin scheduling
  • Buffers in music players and iPod playlists
Quick comparison

Arrays vs Linked Lists

MemoryArray = fixed at declaration. Linked list = allocated at runtime (dynamic).
AccessArray = direct random access. Linked list = must traverse from start.
Insert/DeleteArray = slow (shifting needed). Linked list = fast with right pointer.
SpaceArray = no extra overhead. Linked list = extra pointer per node.

Singly vs Doubly vs Circular

SinglyOne direction, one pointer per node. Simplest.
DoublyTwo directions, two pointers per node (prev + next).
CircularLast node points back to first — no NULL end. Can start from any node.

Overflow vs Underflow

OverflowTrying to INSERT when no memory/space is available. Check before inserting.
UnderflowTrying to DELETE from an empty structure (START = NULL or FRONT = NULL). Check before deleting.

Queue types at a glance

LinearBasic FIFO. Wastes space when elements are removed from front.
CircularFixes wasted space. Rear wraps around using modulo (% MAX).
PriorityPriority determines order, not arrival time (unless same priority).
De-queueInsert/delete at both ends. Most flexible.
Practice questions — tap to reveal answers
+
What is a linked list and how is it different from an array?
A linked list is a sequence of nodes where each node contains data and a pointer to the next node. Unlike arrays, it uses dynamic memory (no fixed size), but unlike arrays it doesn't support direct access — you must traverse from the start. Arrays use contiguous memory; linked lists do not.
+
What are the four types of linked lists?
Singly Linked List (one pointer per node, one direction), Circular Linked List (last node points to first), Doubly Linked List (two pointers per node, traversable both ways), and Header Linked List (always has a header node at the start storing list metadata).
+
What is overflow and underflow? When do you check each?
Overflow = trying to insert when no memory/space is available. Check before every insertion. Underflow = trying to delete from an empty structure (START = NULL or FRONT = NULL). Check before every deletion.
+
What does FIFO mean and which data structure uses it?
FIFO = First In, First Out. The element inserted first is the first to be removed. Queues use FIFO. The first element inserted at the REAR is the first deleted from the FRONT.
+
Why is a circular queue better than a linear queue?
In a linear queue, when elements are removed from the front, those positions become wasted and can't be reused without shifting all elements (expensive). A circular queue connects the rear back to the front using modulo arithmetic (REAR = (REAR+1) % MAX), allowing empty front slots to be reused.
+
How does a priority queue decide which element to process next?
By assigned priority — higher priority is processed first. If two elements have the same priority, they follow FIFO order. In a linked priority queue, each node stores: data, priority number, and the address of the next node.
+
What is a de-queue and how is it different from a regular queue?
A de-queue (double-ended queue) allows insertion and deletion at both ends — either front or rear. A regular queue only inserts at the rear and deletes from the front. De-queues are implemented with circular arrays and maintain LEFT and RIGHT pointers.
+
How is a polynomial represented using a linked list?
Each term of the polynomial becomes one node. The node stores two values: the coefficient and the power/exponent. For example, 10x² + 6x + 9 becomes three nodes: (10, 2) → (6, 1) → (9, 0).
