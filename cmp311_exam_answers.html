Question 1 15 Marks
a) What is a programming language? (2 marks)

A programming language is a formal, structured system of rules (syntax) and meanings (semantics) used to write instructions that a computer can understand and execute. It serves as an intermediary between human programmers and the hardware, allowing developers to express algorithms and logic in a readable and maintainable way. Examples include Python, C, Java, and Assembly.

b) List four programming paradigms and discuss one with an example (10 marks)

The four programming paradigms are:

  1. Procedural (Imperative) Programming
  2. Object-Oriented Programming (OOP)
  3. Functional Programming
  4. Logic (Declarative) Programming

Discussion – Object-Oriented Programming (OOP):

OOP is a paradigm that organises software design around objects rather than functions and logic. An object is an instance of a class, which is a blueprint that defines attributes (data) and methods (behaviours). The four key pillars of OOP are:

  • Encapsulation – Bundling data and the methods that operate on it inside a single unit (a class), and restricting direct access to the internal state.
  • Inheritance – Allowing a new class (child/subclass) to inherit properties and methods from an existing class (parent/superclass).
  • Polymorphism – The ability of different objects to respond to the same method call in different, context-specific ways.
  • Abstraction – Hiding complex implementation details and exposing only the essential features of an object.

Example (Python):

class Animal: def __init__(self, name): self.name = name # Encapsulation def speak(self): # Abstraction pass class Dog(Animal): # Inheritance def speak(self): return self.name + " says Woof!" class Cat(Animal): # Inheritance def speak(self): return self.name + " says Meow!" # Polymorphism animals = [Dog("Rex"), Cat("Whiskers")] for animal in animals: print(animal.speak()) # Output: # Rex says Woof! # Whiskers says Meow!

Here, Dog and Cat both inherit from Animal, yet each overrides speak() differently — this is polymorphism in action.

b) What are the basic concepts of OOPs? Explain three of them (10 marks)

The basic concepts (pillars) of Object-Oriented Programming are: Encapsulation, Inheritance, Polymorphism, and Abstraction. Below are three explained in detail:

1. Encapsulation

Encapsulation is the process of bundling the data (attributes) and the methods that manipulate that data into a single unit called a class. It also involves restricting direct access to some of an object's components by using access modifiers (e.g. private, protected, public). This protects internal data from unintended modification.

class BankAccount: def __init__(self, balance): self.__balance = balance # private attribute def get_balance(self): # controlled access return self.__balance def deposit(self, amount): if amount > 0: self.__balance += amount

2. Inheritance

Inheritance enables a new class (subclass/child) to acquire the properties and methods of an existing class (superclass/parent) without rewriting them. This promotes code reuse and establishes an "is-a" relationship. A subclass can also override inherited methods to provide its own implementation.

class Vehicle: def __init__(self, brand): self.brand = brand def start(self): return "Vehicle starting" class Car(Vehicle): # Car inherits from Vehicle def start(self): # Method overriding return self.brand + " car engine starting"

3. Polymorphism

Polymorphism (meaning "many forms") allows objects of different classes to be treated uniformly through a common interface. A single method name can behave differently depending on the object calling it. There are two types:

  • Method Overriding (Runtime Polymorphism) – A subclass redefines a method from the parent class.
  • Method Overloading (Compile-time Polymorphism) – A class defines multiple methods with the same name but different parameters (supported in Java, C++, but not directly in Python).
class Shape: def area(self): pass class Circle(Shape): def __init__(self, r): self.r = r def area(self): return 3.14159 * self.r ** 2 # Polymorphic behaviour class Rectangle(Shape): def __init__(self, l, w): self.l, self.w = l, w def area(self): return self.l * self.w # Different implementation, same method name

b) Function of Syntactic Analyser and Lexical Analyser (5 marks)

Lexical Analyser (Lexer / Scanner):

The lexical analyser is the first phase of a compiler/interpreter. Its job is to read the source code character by character and group them into meaningful tokens (as shown above). It removes whitespace and comments, identifies keywords, identifiers, operators, literals, and punctuation, and produces a stream of tokens as output for the next phase. If it encounters an illegal character, it reports a lexical error.

Syntactic Analyser (Parser):

The syntactic analyser is the second phase. It takes the stream of tokens from the lexical analyser and checks whether they follow the rules of the programming language's grammar (syntax rules). It builds a Parse Tree (or Abstract Syntax Tree – AST) that represents the hierarchical structure of the program. If the tokens do not conform to the grammar, it reports a syntax error. For example, it checks that parentheses are properly matched, statements are correctly structured, and expressions are valid.

c) List 5 semantic errors (5 marks)

Semantic errors are errors that are syntactically correct but violate the meaning or logic rules of the language. The compiler/interpreter may or may not catch these:

  1. Type Mismatch Error – Attempting to perform an operation between incompatible types, e.g. adding a string to an integer without conversion: int x = "hello" + 5;
  2. Undeclared Variable Error – Using a variable that has not been declared or is out of scope: print(y); where y was never declared.
  3. Division by Zero – Dividing a number by zero at runtime: int result = a / 0;
  4. Wrong Number of Arguments – Calling a function with more or fewer arguments than it expects: defining def add(a, b) but calling add(1, 2, 3);
  5. Incompatible Return Type – A function declared to return an integer but returning a string or nothing: int getAge() { return "twenty"; }
Question 4 15 Marks
a) Explain static binding and dynamic binding (4 marks)

Static Binding (Early Binding):

Static binding is resolved at compile time. The compiler determines which method or variable to use before the program actually runs. This is faster because there is no lookup needed at runtime. It is used with non-virtual functions, overloaded functions, and with statically typed languages like C and Java (for non-polymorphic calls).

Example: In C++, a call to a non-virtual member function is statically bound — the compiler knows exactly which function to call based on the declared type.

Dynamic Binding (Late Binding):

Dynamic binding is resolved at runtime. The actual method or function to execute is determined only when the program is running, based on the actual type of the object (not the declared type). This is essential for implementing polymorphism. It is slightly slower due to the runtime lookup but provides greater flexibility.

Example: In Java, when a parent class reference points to a child class object and a virtual/overridden method is called, Java decides at runtime which child class's method to invoke.

// Dynamic binding example (Java-style pseudocode) Animal a = new Dog(); // declared as Animal, actual type is Dog a.speak(); // resolved at runtime → calls Dog's speak()
b) What is the definition of a control structure? (2 marks)

A control structure is a programming construct that determines the order in which statements or instructions are executed. Instead of executing code strictly top-to-bottom (sequential flow), control structures allow programs to make decisions, repeat actions, or jump to different parts of the program based on conditions. The three main types are:

  • Sequential – Statements execute one after another in order.
  • Selection (Conditional) – Executes different code paths based on a condition (e.g. if-else, switch).
  • Repetition (Loops) – Repeats a block of code multiple times (e.g. while, for, do-while).

a) Parse Tree using Left-most Derivation for "aabcbe" (8 marks)

Given Grammar (Production Rules):

S → aABe A → Abc | a B → d

Input string: "aabcbe"

Note: Looking at the input string "aabcbe", we observe it does not contain the character 'd', which is the only production for B. However, examining the structure carefully, the intended input that matches this grammar would be "aabcde" (with 'd'). We will derive "aabcde" as this is consistent with the grammar. If the input truly is "aabcbe", it cannot be generated by this grammar.

Left-most Derivation (always expand the leftmost non-terminal first):

Step 1:   S  ⇒  aABe              (applying S → aABe)
Step 2:   aABe  ⇒  aAbcBe       (applying A → Abc — leftmost A)
Step 3:   aAbcBe  ⇒  aabcBe     (applying A → a — leftmost A)
Step 4:   aabcBe  ⇒  aabcde     (applying B → d)
Final string: a a b c d e ✓

Parse Tree:

S / | \ a A B e / | \ | A b c d | a

Reading the leaf nodes left to right: a – a – b – c – d – e, which gives us the string "aabcde".

Detailed Parse Tree Breakdown:

  • Root node S produces: a, A, B, e (via S → aABe)
  • First A (child of S) produces: A, b, c (via A → Abc)
  • Second A (child of the first A) produces: a (via A → a) — this is a terminal
  • B (child of S) produces: d (via B → d) — this is a terminal
b) Explicit vs Implicit Type Conversion with implementations (7 marks)

Implicit Type Conversion (Automatic/Coercion):

Implicit conversion happens automatically by the compiler or interpreter when it converts a value from one type to another without the programmer explicitly requesting it. It usually converts from a "smaller" or "narrower" type to a "larger" or "wider" type to avoid data loss (e.g. int → float).

Explicit Type Conversion (Casting):

Explicit conversion is done manually by the programmer using casting syntax or conversion functions. It is required when the conversion cannot be done automatically or when converting from a wider type to a narrower type (which might lose data, e.g. float → int).

a) What are Abstract classes and Abstract methods? (5 marks)

Abstract Class:

An abstract class is a class that cannot be instantiated (i.e. you cannot create objects directly from it). It is designed to serve as a base class for other classes. It may contain both abstract methods (without implementation) and concrete methods (with implementation). The purpose of an abstract class is to define a contract — a set of methods that all subclasses must implement.

Abstract Method:

An abstract method is a method that is declared in an abstract class but has no body/implementation. Subclasses are required to override and provide their own implementation of abstract methods. If a subclass does not implement all abstract methods, it also becomes abstract and cannot be instantiated.

Example (Python using ABC):

from abc import ABC, abstractmethod class Shape(ABC): # Abstract Class @abstractmethod def area(self): # Abstract Method – no body pass def describe(self): # Concrete Method – has implementation return "I am a shape" class Circle(Shape): # Concrete subclass def __init__(self, radius): self.radius = radius def area(self): # Must override abstract method return 3.14159 * self.radius ** 2 # shape = Shape() # ERROR: Cannot instantiate abstract class circle = Circle(5) print(circle.area()) # Output: 78.53975 print(circle.describe()) # Output: I am a shape

Key Points: Abstract classes enforce a structure on subclasses; they promote code reuse through concrete methods; and abstract methods ensure a consistent interface across all subclasses.

b) What paradigm(s) does Python belong to? (2 marks)

Python is a multi-paradigm programming language. It supports the following paradigms:

  • Procedural (Imperative) Programming – Python allows writing code as a sequence of instructions using functions.
  • Object-Oriented Programming (OOP) – Python supports classes, inheritance, encapsulation, and polymorphism.
  • Functional Programming – Python supports first-class functions, lambda expressions, map, filter, reduce, and list comprehensions.

Because Python supports all these paradigms, it is classified as a multi-paradigm language, giving developers the flexibility to choose the style that best fits their problem.

d) Explain the parameter passing technique in the C code snippet (5 marks)

Given Code:

void swapnum(int* i, int* j) { int temp = *i; *i = *j; *j = temp; }

Technique: Pass by Pointer (Pass by Reference using Pointers)

This function uses Pass by Pointer (also called Pass by Reference in the context of C), which is a variation of pass-by-value where the addresses (memory locations) of variables are passed instead of their values. Here is how it works step by step:

  • int* i, int* j – The parameters i and j are pointers. They store the memory addresses of the variables passed by the caller.
  • *i – The asterisk (*) is the dereference operator. It accesses the value stored at the address that i points to.
  • int temp = *i; – Stores the value of the variable that i points to in a temporary variable.
  • *i = *j; – Replaces the value at address i with the value at address j.
  • *j = temp; – Replaces the value at address j with the saved temporary value.

Why Pass by Pointer? In C, there is no true "pass by reference" like in C++ or Java. Instead, pointers are passed by value (the address itself is copied), but since the function receives the address of the original variables, it can modify the original variables directly. This is essential for functions like swapnum that need to change the caller's data.

Calling Example:

int x = 10, y = 20; swapnum(&x, &y); // & is the address-of operator // After the call: x = 20, y = 10

The & operator passes the addresses of x and y, allowing swapnum to swap their values in-place.
